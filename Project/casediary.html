E-case Diary (Advocate Case Management)

E-case Diary is a web application designed to streamline the management of legal cases for advocates. It provides a centralized platform for advocates to organize and track their cases, manage client information, and schedule important dates and deadlines. The application offers features such as case categorization, document storage, and task management, allowing advocates to efficiently handle their caseload and improve productivity. With E-case Diary, advocates can easily access case details, collaborate with clients and colleagues, and stay organized throughout the legal process.

1. Login Page

This page allows advocates to log in to their E-case Diary account using their credentials. User enters their username and password to access their account. and password and username verified from the database using JDBC. if user credentials are valid, they are redirected to the dashboard page. otherwise, an error message is displayed.

2. Registration Page

If a user is new to the system, they can register by providing their details. The registration process involves entering personal information, setting up a username and password, and submitting the form. Once registered, the user is directed to the login page where they can access their account.

3. Dashboard Page

After successful login, users are directed to the dashboard page. The dashboard displays an overview of all cases assigned to the advocate, including case status, and recent activities. Users can also add new cases, update existing case details, and view client information from this page. it displays total case , solve case and pending case. And also After click on logout the session is destroyed and user is redirected to the login page.

4. Delete Case

This section allows advocates to delete cases from their E-case Diary. Users can select a case and confirm deletion. Once confirmed, the case is removed from the system and cannot be recovered.

5. Pending Case

This section displays all pending cases assigned to the advocate. Users can view details of each pending case, update their status, and manage deadlines.

6. Solved Case

This section displays all solved cases assigned to the advocate. Users can view details of each solved case and track their resolution status.

Technologies Used

HTML, CSS, JSP Servlet, JDBC, MySQL
